# baza_bot — Telegram-ассистент для партнёров BAZA Development

Бот отвечает агентам по недвижимости на вопросы по базе знаний ЖК: отделка, характеристики, вознаграждения, финансовые инструменты, контакты менеджера. Текст (Часть 1) и голос (Часть 3). Честно говорит «не знаю» вместо выдумок.

Бот в Telegram: **@baza_parthners_bot**

---

## Стек

Nuxt 4 + TypeScript · Prisma 7 (+ `@prisma/adapter-pg`) · PostgreSQL 17 + pgvector · grammY · Playwright · bothub (LLM + эмбеддинги) · Soniox (STT).

---

## Быстрый запуск (локально)

```bash
# 1. Зависимости и БД
pnpm install
docker compose up -d db                    # Postgres+pgvector на порту 5433

# 2. Секреты
cp .env.example .env                        # вписать BOTHUB_API_KEY, TELEGRAM_BOT_TOKEN

# 3. Схема + наполнение базы знаний
pnpm prisma migrate deploy
pnpm ingest                                 # тянет 17 источников + портал, ~2 мин
                                            # при 503 от Google/Yandex — повторить (идемпотентно)

# 4. Бот (long polling)
pnpm bot
```

## Деплой (домашний сервер, Docker)

```bash
# .env на сервере: POSTGRES_PASSWORD, BOTHUB_API_KEY, TELEGRAM_BOT_TOKEN
docker compose -f docker-compose.prod.yml up -d db
docker compose -f docker-compose.prod.yml run --rm migrate
docker compose -f docker-compose.prod.yml run --rm ingest    # headed chromium под xvfb
docker compose -f docker-compose.prod.yml up -d bot
```

Long polling — публичный HTTPS/webhook не нужен, бот работает из-за NAT.

## Архитектура поиска по знаниям — и почему именно она

**Дёшево и быстро** (критерий ТЗ) достигается двухуровневой маршрутизацией, а не одним RAG:

```
вопрос
 ├─ 1. интент (keyword) → canned из реестра        0 LLM, ~0 центов
 │     вознаграждение (из fee), ипотека/рассрочка/фиксация (ссылка по региону),
 │     сделка (КЦ), контакты (менеджер по региону)
 ├─ 2. проект (alias-роутер по основам слов)
 │     ├─ агентство → «BAZA реализует как агентство, КВ 1%» + менеджер
 │     ├─ Бали → вне области + менеджер
 │     └─ застройщик → корпус проекта → gpt-4.1-nano
 └─ 3. проект не назван → pgvector top-k по всей базе
```

**Приоритет интента над проектом:** «Сколько первоначалка на ДМД2» → ссылка на рассрочку (canned), а не пересказ корпуса.

### Чем выбор обоснован — замеры на живых API

База знаний — 642 чанка, ~55k токенов. Сравнили 4 архитектуры на одном честном индексе:

| Вопрос | Full-context 69k (0.70¢) | Наивный RAG top-6 (0.013¢) | RAG+реранк (0.04¢) | **Project-scoped (выбран)** |
|---|---|---|---|---|
| Отделка Алисы | ❌ приплёл «Скандинавию» (Бестселлер) | ❌ потерял White Box, Урбан-лофт | ⚠️ частично | ✅ все варианты |
| ДМД2 этажность | ❌ **выдумал потолки** из ЖК ДНК | ❌ «со 2 по 16 этаж» | ❌ регресс | ✅ верно + честно «потолки не указаны» |
| Отделка Бестселлера | ❌ приплёл «Урбан-лофт» (Алиса) | ❌ неполно | ✅ | ✅ верно |

- **Full-context отвергнут по точности, не по цене:** на 69k токенов модель смешивает проекты и выдумывает числа — прямое нарушение «выдуманные цены = минус».
- **Наивный RAG** проваливает 3 из 4: соседние разделы вытесняют нужный чанк.
- **Реранк** чинит одно, ломает другое — нужный чанк не доходит до top-30.
- **Project-scoped:** роутинг детерминирован по алиасам заказчика (не по вероятностному score). Малый корпус ЖК (ДМД2 ~4k токенов) отдаётся целиком; крупный (Алиса, Бестселлер) сужается vector top-k внутри проекта.

**Цена и задержка (без кэша):** canned — 0¢; малый проект — 0.02–0.06¢, 2.3–3s; крупный — 0.05–0.26¢, 2.5–5.3s. При лимитах ТЗ (5¢, 30s) — запас 19× и 6×.

**Источник истины по условиям — партнёрский портал** `realtors.baza.bz` (комиссии, рассрочка), а не таблица тест-кейсов: портал свежее и опубликован. Комиссия — структура (у Алисы 2.6% + 1% на выбор себе или скидкой клиенту), не число.

### Повторяемый ingest

`pnpm ingest` тянет из публичных URL (сайты через headed Playwright — headless блокируется антиботом; Google export; Yandex Disk; обход дерева портала). Идемпотентно по content-hash: перезапуск не дублирует. Structure-aware чанкинг (лист xlsx / страница pdf / блок Tilda = раздел) — наивный чанкинг проваливал тест на отделку.

## Часть 2. Админка (shadcn-vue)

Маршрут `/admin` (`app/pages/admin/`), три страницы + 4 Nitro-роута в `server/api/admin/`:

- **Источники** (`/admin`) — список источников базы знаний (URL сайтов, документы), статус последнего парсинга, кнопка **«Переобновить»** (запускает ingest, поллинг статуса).
- **База знаний** (`/admin/knowledge`) — просмотр того, что лежит в базе: чанки по проектам, фильтр + поиск, пагинация.
- **Диалоги** (`/admin/dialogs`) — история: кто спросил, что ответил бот.

Компоненты берутся из дизайн-системы как есть, без кастомного дизайна.

**Про стиль.** ТЗ указывает `shadcn/ui, style: default, baseColor: neutral`. `baseColor: neutral` соблюдён. Стиль `default` в порте **shadcn-vue депрекейтнут** (после `init` переключить необратимо) — актуальная версия оставляет только `new-york`, поэтому в `components.json` → `"style": "new-york"`. Это единственное отступление от буквы ТЗ, вынужденное ограничением инструмента.

## Часть 3. Страница из макета Figma (через Figma MCP)

Экран **«Главный экран»** приложения партнёра (Figma node `932:22716` файла Elvikom) собран на маршруте **`/elvikom`** — `app/pages/elvikom.vue`, собственный layout (`layout: false`), Tailwind v4 + scoped-CSS, токены дизайн-системы в CSS-переменных. Перенос сделан **через Figma MCP** (`get_design_context`), а не руками по скриншоту.

**Как отработал MCP:**

- **Хорошо.** `get_design_context` одним вызовом отдал React+Tailwind-референс, скриншот фрейма и ссылки на все ассеты. Дизайн-токены пришли точными значениями: палитра (`--01 #e20617` … `--07 #d8e6f4`), шрифты **Inter**/**Unbounded**, тени `#0023460F`, `letter-spacing −0.4px`, радиусы 20/44px. `get_metadata` дал дерево фреймов — по нему выбран самый наполненный из шести вариантов «Главного экрана».
- **Что правил руками.**
  - Референс на React+Tailwind — конвертирован в Vue SFC (structure/spacing/typography сохранены, классы → scoped-CSS + токены).
  - **Логотип BAZA** — два SVG с `preserveAspectRatio="none"`; при `position:absolute` через `inset` тег `<img>` брал intrinsic-высоту и растягивался. Заменил на flex-стек с явными габаритами долей 45×33.
  - Иконка `ellipsis.circle` пришла плейсхолдером `<SFSymbol>` — заменена на lucide `Ellipsis` в кружке.
  - iOS-статусбар: SF Pro не встроен → время рендерится системным шрифтом; батарея собрана из div'ов (в макете — векторные примитивы, а не один ассет).
  - Ассеты (3D-иконки — PNG; QR, лого, статусбар — SVG) скачаны в `public/figma/`: MCP-ссылки на ассеты живут 7 дней, держим локально.
- **Лимиты.** Аккаунт Figma **starter / seat View** → ~6 MCP read-вызовов в месяц. Страница собрана за **3 вызова**: 2× `get_metadata` (обзор дерева) + 1× `get_design_context` (код+скриншот+ассеты).

Шрифты самохостятся (`@fontsource-variable/inter`, `@fontsource-variable/unbounded`) — без рантайм-зависимости от Google Fonts.

## Что не вошло (осознанно)

- **Бали** (Origins, Kedungu) — есть в справочнике заказчика, но вне тем ТЗ и таблицы вознаграждений. Бот распознаёт вопрос и честно отвечает «вне моей области» + менеджер.
- **Сканы** без текстового слоя (`Инфраструктура_ДНК.pdf` и др.) — помечаются `NEEDS_OCR`, не дают пустых чанков. Их содержание дублируется другими источниками.
