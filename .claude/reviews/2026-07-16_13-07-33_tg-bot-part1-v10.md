## BLOCKING Findings

### B1. Существует `realtors.baza.bz/msk` — план изобрёл ответ там, где у заказчика есть готовая страница
- **Dimension:** D10 (Input Quality Audit)
- **Where:** Реестр → «**Рассрочка** → `realtors.baza.bz/ekb` — **только ЕКБ**. Для Бестселлера условия рассрочки в docx «Инструменты продаж» (50/50 без удорожания, ПВ 50%) — контент, не ссылка»
- **Problem:** `/msk` отвечает 200 и содержит: «Финансовые инструменты BAZA Development **в Москве** … Способы покупки: Ипотека, Траншевая ипотека, **Рассрочка**, Трейд-ин». Это прямой симметричный аналог `/ekb`. План вместо него конструирует ответ из docx — то есть отвечает своими словами там, где заказчик даёт официальную страницу. Плюс `/msk` живая и актуальная, а docx датирован «с 01.12.2025».
- **Fix required:** рассрочка → `/ekb` для ЕКБ, **`/msk` для Москвы**. Симметрично, без изобретений.

### B2. Комиссия по Бестселлеру опубликована — план объявил её неизвестной
- **Dimension:** D10
- **Where:** Реестр → «Бестселлер … КВ **нет в таблице**»; «**Вознаграждение** → таблица покрывает 10 проектов, **Бестселлера в ней нет** → для него менеджер МСК»; acceptance → «"какой процент КВ по Бестселлеру" → **не таблица** → менеджер Дружинина Алёна»
- **Problem:** На `/msk` дословно: **«Агентская комиссия 2,1%»**. Вывод v7 («Бестселлера нет в таблице вознаграждений») был верен — но из него сделан неверный вывод, что данных нет вообще. Их просто нет **в таблице**; они на партнёрской странице.
- **Blast radius:** это acceptance-критерий, закрепляющий **отказ отвечать на вопрос, ответ на который существует и опубликован**. Четвёртый повтор паттерна: v1 — «не знаю» про ДМД2 при живом источнике; v3 — ожидание из галлюцинации; v9 — сверка с неполным эталоном; теперь — «спроси менеджера» вместо «2,1%».
- **Fix required:** `feePercent` Бестселлера = 2,1% (источник — `/msk`); критерий переписать на выдачу цифры.

### B3. Точка входа для фиксации существует — решение владельца принято на неполном отчёте
- **Dimension:** D10 / D6
- **Where:** Реестр → «**Фиксация → перевод на менеджера.** Ссылки на раздел в миниаппе нет ни в одном из 17 источников»
- **Problem:** На `/ekb` — кнопка **«Закрепить клиента»** и текст «Все в одном месте! **Фиксация клиентов.** Расписание мероприятий. Материалы по проектам. ИИ-помощник. **Макс Телеграм**». В разметке — ссылки на `max.ru/id6685156649_2_bot` и партнёрские каналы. То есть «миниапп» из ячейки — это бот в Max/Telegram, а точка входа — партнёрская страница.
- **Почему BLOCKING:** владелец задачи выбрал «перевод на менеджера» **на основании моего отчёта**, что ссылки не существует. Отчёт был неполон: я искал в корпусе, но не открыл страницу, на которую ссылается соседний canned-ответ. Решение надо пересмотреть на полных данных — это не мой выбор.
- **Fix required:** предъявить владельцу найденное и переспросить.

Правки не косметические: меняется реестр (feePercent, ссылки по регионам), список источников (+2 страницы), canned-тексты, и два решения требуют пересмотра владельцем. `BLOCK`.
